# CyberLab ContentOps 需求分析

更新日期：2026-06-16

## 1. 项目背景

CyberLab ContentOps 是一个课程实验型内容运营控制台，用于演示和实训“趋势/内容驱动、多平台分发”的内容运营链路。项目需要同时满足教学演示、真实工具接入和安全审计的要求。

## 2. 总体目标

1. 支持从互联网（RSS、公开榜单）采集优质内容源。
2. 支持使用真实 LLM 对内容进行分析、摘要和多平台改写。
3. 支持为小红书、今日头条、微信公众号生成 preview/draft 载荷。
4. 支持真实调用外部发布工具，但默认停留在预览/草稿/手动确认阶段，不自动点击发布。
5. 支持外部工具仓库的 GitHub 同步状态检查。
6. 提供可审计的运行日志与持久化存储。

## 3. 用户角色与用例

| 角色 | 典型用例 |
|------|----------|
| 学生 | 选择趋势/RSS 文章 → LLM 分析 → 生成三平台版本 → 查看预览载荷 → 观察外部工具执行结果 |
| 教师 | 查看运行日志、导出实验报告、检查学生是否遵守安全预览模式 |
| 运维/课程管理员 | 配置 OpenCode API key、同步外部工具仓库、监控平台适配器状态 |

## 4. 功能需求

### 4.1 内容采集

- 接入 BestBlogs WeChat2RSS OPML，并发抓取约 190 个公众号 RSS 源。
- 保留原有公开榜单（Hacker News、知乎、微博等）采集能力。
- 支持按关键词、时间范围、源站进行筛选。
- 采集失败时保留可演示的 fallback 数据或空列表，不影响主流程。

### 4.2 内容分析与生成

- 调用 OpenCode 免费模型对 RSS 文章进行摘要、标签、受众、适合平台、调性分析。
- 支持模板生成与 LLM 生成两种模式，LLM 失败时自动回退模板。
- 输出小红书、今日头条、微信公众号三个平台版本。

### 4.3 发布管理

- 每个平台提供“生成预览载荷”和“执行真实预览”两个动作。
- 默认安全模式：不自动点击发布按钮、不自动群发。
- 小红书：调用 `publish_pipeline.py --preview` 填充表单。
- 今日头条：调用 `toutiao_cli` 校验 Markdown 并给出手动确认命令。
- 微信公众号：调用 `publish.mjs` 创建草稿（受 `WEMP_ALLOW_DRAFT` 开关控制）。

### 4.4 外部工具仓库管理

- 检查 `external/toutiao`、`external/XiaohongshuSkills`、`external/wemp-operator` 的 git 状态。
- 支持一键 `git pull` 同步。

### 4.5 持久化与审计

- 使用 SQLite 持久化趋势、文章、RSS 源、草稿、日志，重启不丢失。
- 所有关键动作产生 JobEvent，包含类型、标题、消息、状态、时间。

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 安全 | 平台 Cookie、AppSecret、API key 不写入代码与 Git；真实发布需人工确认 |
| 可审计 | 每次外部动作保留命令摘要、stdout/stderr 摘要、时间戳 |
| 可替换 | RSS 源、LLM、平台适配器均通过接口抽象，便于替换 |
| 可维护 | 后端分层清晰，前端视图独立，新增平台只需增加适配器 |
| 兼容性 | 后端 Python 3.9+，前端现代浏览器，外部工具在 macOS 上可验证 |

## 6. 边界与假设

- 外部平台登录态由用户在本地手动完成（扫码或配置）。
- OpenCode 免费模型可能限流，系统需支持无 LLM 的模板回退。
- 微信公众号草稿创建属于真实副作用，默认关闭，仅用于测试号。
- 小红书 CDP 预览会打开 Chrome 窗口，适合本地演示。
