# 开发计划与验收标准

更新日期：2026-06-02

本文档用于伴随开发，记录 CyberLab ContentOps 从课程实验脚手架演进到可审计平台的计划、验收标准和风险控制要求。

## 1. 当前基线

当前系统已经具备：

- FastAPI 后端与 React/Vite 前端。
- 趋势采集入口，支持外部脚本与备用趋势。
- 三平台内容生成模板。
- 小红书、今日头条、微信公众号的预览适配器。
- 发布管理页面和运行日志页面。
- 后端 pytest 与前端 Vitest 基础测试。

当前系统尚未具备：

- 持久化任务和草稿记录。
- 真实 LLM 内容生成。
- 真实平台发布或草稿创建。
- 用户、班级、小组或账号隔离。
- 完整端到端审计报告导出。

## 2. 开发原则

1. 预览优先：新增平台动作必须先实现 preview 或 draft-only。
2. 人工审核优先：真实发布必须由人工确认，不允许默认自动发布。
3. 可审计优先：每个外部动作都要能追踪输入、输出、时间、状态和失败原因。
4. 可替换优先：爬虫、生成器、平台适配器都应通过接口抽象扩展。
5. 课程可用优先：外部依赖失败时，系统仍应通过备用数据支撑课堂演示。
6. 等级推进优先：新增能力先在 `capability_levels.md` 登记等级，再按验收标准升级。
7. 指标闭环优先：关键运行问题应沉淀为稳定审计指标，而不是只靠日志文本排查。

## 3. 阶段计划

| 阶段 | 模块 | 能力目标 | 实现说明 | 验收标准 | 审计材料 |
| --- | --- | --- | --- | --- | --- |
| P1 | 趋势源注册表 | 可添加、禁用和标记数据源健康状态 | 将源站元数据从内存样例中抽出，形成 registry | `/api/trends/collect` 可跳过禁用源并记录原因 | 源站清单、采集日志、失败样例 |
| P1 | 任务事件存储 | 重启后保留运行日志 | 用 SQLite 替换内存 job 存储 | 重启后 `/api/jobs` 仍能看到历史事件 | 数据库迁移说明、持久化测试 |
| P1 | 内容生成契约 | 支持模板生成器和 LLM 生成器切换 | 定义 `ContentGenerator` 接口和 provider 配置 | 单元测试证明模板和 mock LLM 输出同一 `PostDraft` 结构 | 生成器配置、提示词版本、测试记录 |
| P2 | 小红书适配器 | 接入 CDP 预览流程 | 包装 `publish_pipeline.py --preview` | 返回命令、载荷、截图或日志，不自动发布 | 预览截图、命令参数、风险确认 |
| P2 | 今日头条适配器 | 接入 MCP/HTTP 或 CLI 预览桥 | 优先包装手动确认式预览，再修复 MCP 服务 | mock MCP 测试验证 `publish_micro_post` 载荷 | 载荷样例、登录态要求、失败路径 |
| P2 | 微信公众号适配器 | 生成本地 Markdown 草稿 | 将文章版内容转为 wemp 兼容 Markdown | 后端返回草稿路径和命令提示 | 草稿文件、字段映射、素材需求 |
| P3 | 审核工作流 | 增加人工审批状态 | 增加 generated、previewed、reviewed、approved、rejected 等状态 | 未 approved 时 UI 禁止真实动作 | 状态转换表、操作人、时间戳 |
| P3 | 素材管线 | 管理封面、配图提示词和生成资产 | 保存 image prompt、资产路径和来源 | 每个草稿可追踪素材来源与生成状态 | 资产清单、提示词、版权说明 |
| P4 | 实验分析 | 输出课程实验报告 | 记录热点、草稿、预览、审核和日志 | 教师可导出一次实验完整摘要 | 报告样例、评分维度 |
| P4 | 用户与账号 | 支持学生或小组隔离 | 增加账号模型和权限依赖 | 两个工作区数据互不可见 | 权限测试、账号配置说明 |

## 4. 近期开发顺序

建议近期按以下顺序推进：

1. 增加 SQLite 持久化，先保证日志和草稿不会因重启丢失。
2. 抽象内容生成器接口，为后续 LLM 接入保留稳定契约。
3. 引入机器可读审计指标结构，至少覆盖采集 fallback、生成失败、适配器预览和真实动作阻断。
4. 给今日头条、小红书、微信适配器增加 mock 集成测试。
5. 接入小红书和今日头条的 preview-only 命令包装。
6. 增加审核状态，阻止未审核内容进入真实发布路径。
7. 增加实验报告导出，形成课程闭环。

## 5. 完成定义

每个功能合并或交付前应满足：

- 有清楚的输入、输出和失败路径。
- 有后端 API 测试、服务测试或前端组件测试。
- 可见 UI 覆盖加载中、成功、空状态和失败状态。
- 涉及页面变化时完成浏览器截图或交互检查。
- 涉及外部平台时保持 preview-only 或明确审核门禁。
- 涉及新平台、数据源、模型或命令时更新 `docs/` 文档。
- 涉及模块能力升级时更新 `capability_levels.md` 和 `validation_protocol.md`。
- 涉及运行信号变化时更新 `audit_metrics.md`。
- 不提交账号、Cookie、AppSecret、验证码、真实用户隐私数据。

## 6. 测试策略

| 层级 | 当前工具 | 适用范围 | 后续要求 |
| --- | --- | --- | --- |
| 后端单元与契约 | pytest | API 响应模型、服务逻辑、适配器载荷 | 新增平台适配器必须有 mock 测试 |
| 前端组件 | Vitest 与 Testing Library | 页面渲染、交互状态、核心文案 | 新增视图必须覆盖主要用户动作 |
| 浏览器检查 | Playwright 或内置 Browser | 导航、响应式、控制台错误、截图 | 可见工作流变化需要截图证据 |
| 外部工具检查 | mock subprocess 或 mock HTTP client | 外部脚本命令和载荷 | 默认不做真实发布，不保存敏感登录态 |
| 端到端实验 | 后续补充 | 从采集到报告导出 | 使用测试账号和沙箱数据 |

## 7. 风险登记

| 风险 | 影响 | 当前控制 | 后续动作 |
| --- | --- | --- | --- |
| 平台 DOM 改版 | 自动化发布脚本失效 | 先保持预览模式 | 保存截图和选择器版本 |
| 平台风控 | 限流、封号或内容拦截 | 课程演示不自动发布 | 使用测试号、限速、人工审核 |
| 数据源不可用 | 热点采集失败 | 使用备用趋势 | 增加源站健康状态 |
| LLM 输出不可控 | 内容质量和合规风险 | 当前使用模板 | 增加审核、提示词版本和输出检测 |
| 状态不持久 | 重启后记录丢失 | 当前只适合演示 | P1 增加 SQLite |
| 密钥泄露 | 账号与平台安全风险 | 文档禁止记录敏感信息 | 增加配置模板和密钥扫描 |

## 9. 文档闭环

开发任务与文档的对应关系：

| 开发内容 | 必须更新 |
| --- | --- |
| 新增模块、拆分服务、改变调用边界 | `architecture.md` |
| 新增或升级平台接入 | `platform_status.md`、`capability_levels.md`、`validation_protocol.md` |
| 新增审计信号或日志结构 | `audit_metrics.md`、`audit_guide.md` |
| 新增实验报告、截图、预览文件 | `experiment_artifacts.md` |
| 新增命令、脚本或 Makefile target | `commands.md` |
| 调整阶段排期或验收标准 | `development_plan.md` |
