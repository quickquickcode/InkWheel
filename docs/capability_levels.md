# 能力等级表

更新日期：2026-06-16

本文档跟踪 CyberLab ContentOps 中各类能力的成熟度。

## 1. 等级定义

| 等级 | 名称 | 判定标准 |
| --- | --- | --- |
| L1 | 基础可用 | 能跑通 happy path，主要依赖人工判断，失败信息较粗。 |
| L2 | 结构化可用 | 输入输出结构清楚，有基础测试，有明确失败路径。 |
| L3 | 可审计模块 | 有稳定状态、事件日志、mock 测试和文档记录。 |
| L4 | 强能力模块 | 支持多策略/多来源/多平台，有机器可读审计指标和验收包。 |
| L5 | 闭环模块 | 自动评估、人工审核、回放证据、报告导出和回归验证齐备。当前暂无模块达到 L5。 |

## 2. 趋势源能力

| 模块 | 当前等级 | 能力说明 | 下一步 |
| --- | --- | --- | --- |
| `wemp_fetch_news` | L2 | 后端可通过 `TrendService` 调用 `external/wemp-operator/scripts/content/fetch_news.py`，失败时返回备用趋势。 | 增加源站健康状态、采集耗时、失败原因和稳定审计指标。 |
| `fallback_trends` | L2 | 外部采集不可用时提供确定性课程样例，保证 UI 可演示。 | 在运行日志中明确标记 fallback，并在实验报告中区分真实采集与备用数据。 |

## 7. 升级规则

- 从 L1 升 L2：必须有结构化输入输出和基础测试。
- 从 L2 升 L3：必须有日志、mock 测试和文档记录。
- 从 L3 升 L4：必须有机器可读审计指标、验收包和浏览器/接口验证。
- 从 L4 升 L5：必须能导出完整实验报告，并能基于审计结果进行修复或回放。

## 8. 当前优先级

1. 为三个平台适配器补齐 mock subprocess 测试，稳定在 L3。
2. 建立 `AuditMetric` 结构，覆盖 RSS 采集、LLM 生成、外部工具执行。
3. 接入 `toutiao_scraper` 作为补充趋势源。
4. 实现审核状态机，阻止未 approved 内容进入真实发布路径。
5. 实现实验报告导出，形成课程闭环。
